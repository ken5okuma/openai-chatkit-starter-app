# AIニュースキュレーター

自分専用のAIニュースキュレーター。Web UI(`npm run dev` → http://localhost:3000)と、
チャット操作用のCLI(`scripts/curator.ts`)の両方がある。

## モバイル(Claude Codeアプリ)での操作方法

ユーザーがスマホのClaude Codeからこのリポジトリを開いている場合、あなたがキュレーターの
操作係になる。以下のフローで対応すること。

### ユーザーが「今日のニュース」を求めたら

1. 初回のみ: `npm install`(node_modulesがない場合)
2. `npx tsx scripts/curator.ts today` を実行
3. 出力された記事一覧を読みやすく整形して表示する(番号・タイトル・リンク・要約・タグ)。
   「番号で評価を教えてください(例: 1,3は役に立った / 2は不要)」と添える
4. 新しいバッチを収集した場合(data/ に変更がある場合)はコミットしてプッシュする

### ユーザーが評価を伝えたら

1. `npx tsx scripts/curator.ts feedback <番号(カンマ区切り)> <helpful|skip>` を実行
   - 「役に立った」「いいね」「👍」→ helpful / 「不要」「いらない」「👎」→ skip
   - 評価の取り消しは clear
2. 全件の評価が終わったら `npx tsx scripts/curator.ts prefs` で嗜好プロファイルを見せる
3. **必ず data/ の変更をコミットしてプッシュする**(これが学習データの保存にあたる。
   プッシュを忘れるとセッション終了時に評価が消える)

### 注意事項

- `data/curator/` は嗜好プロファイルと日次バッチの保存場所。**意図的にGit管理している**
  (使い捨てのクラウドセッション間で学習を引き継ぐため)。削除・gitignore追加をしない
- 作業開始時は `git pull` で最新のdata/を取り込むこと
- 出力に「デモデータ」と表示された場合はフィードが取得できていない。この環境の
  ネットワーク設定でRSSフィードのドメインが許可されていない旨をユーザーに伝える
- 要約・タグ付けは `claude -p`(headless)→ キーワード判定の順で自動フォールバックする

## アーキテクチャ

- `lib/curator/` — 収集(rss.ts)・要約タグ付け(tagging.ts)・スコアリング(scoring.ts)・
  選定と重み更新(curator.ts)・永続化(store.ts)
- スコア = 新着性 + ソース信頼度 + タグ嗜好 + ソース嗜好。1日12本(うち2本は探索枠)
- フィードバックでタグ±0.2 / ソース±0.15、範囲は -2〜+2
- 情報源の追加・削除は `lib/curator/sources.ts`
